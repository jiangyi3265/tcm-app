# 医师排班与预约/人员扩展设计

## 范围

本轮一次性覆盖以下四块能力：

1. 医师按周固定排班，半小时粒度，前台与病人预约都基于排班和已预约信息选时段。
2. 人员管理补充医师资料、服务权限、排序号、家庭住址、所属组织/编号。
3. 实习生维护只读实习日期列表，实习日期起 3 天内可只读浏览病历；过期自动失效。
4. 提供匿名对外预约页；未指定医师时，系统按医师排序号自动匹配在岗且可提供该服务的医师。

## 已确认与默认假设

- 排班为每周固定模板，不支持按具体日期例外停诊/加班。
- 时间粒度固定为 30 分钟。
- 服务时长仍取 `serviceTypes[key].duration`。
- 未指定医师时，按医师资料里的排序号从小到大匹配。
- 对外预约无需登录。
- 对外预约遇到新病人时自动建档。
- 医师可提供服务采用多选 `serviceKey` 列表；收费和自动匹配都按该权限过滤。
- 实习日期记录只保存日期；权限窗口为“该日期 00:00 起算的 3 天内”。

## 数据设计

### 用户资料

继续复用 `sys_user.remark` JSON，避免新增表。新增字段：

- `practitionerSortOrder`: 数字，自动分配时的优先级。
- `workingHours`: `{ monday: [{start,end}], ... }`，每个 weekday 是时间段数组。
- `serviceKeys`: `string[]`，医师可提供的服务。
- `internshipDates`: `string[]`，格式 `YYYY-MM-DD`。
- `regulatoryBody`
- `registrationNumber`
- `homeAddress`
- `title`

说明：

- 现有单段 `workingHours[day] = {start,end}` 数据在读取时兼容转换为数组。
- 非医师角色可以不配置 `workingHours / serviceKeys / practitionerSortOrder`。

### 病人档案

继续复用 `tcm_patient.payload`，新增员工自动建档标记：

- `linkedUserId`
- `isStaffProfile`

创建/更新员工时，自动创建或同步对应病人档案，满足“工作人员自动建档”。

## 后端设计

### 1. 用户资料扩展

- 扩展 `TcmUserController` 的 `create / update / list / toMap`。
- 读取和写入 remark JSON 时支持新字段。
- 增加“实习签到”快捷接口：为指定实习生追加当天日期到 `internshipDates`。
- 用户创建/更新后同步员工病人档案。

### 2. 排班与可预约时段

在预约服务层补充统一的可预约计算逻辑：

- 指定医师：
  - 读取该医师周排班。
  - 生成某天半小时槽位。
  - 结合服务时长、现有预约、诊室冲突、预约间隔过滤可用起始时间。
- 未指定医师：
  - 先按服务权限与排序号筛选候选医师。
  - 对每个候选医师计算可用槽位。
  - 汇总成时间槽，附带 `availablePractitionerIds` 与默认 `assignedPractitionerId`。

新增接口：

- 内部：`GET /api/appointments/availability`
- 公开：`GET /api/public-booking/options`
- 公开：`GET /api/public-booking/availability`
- 公开：`POST /api/public-booking`

### 3. 预约创建

- 内部预约允许 `practitionerId` 为空。
- 若为空，后端按排序号自动分配。
- 创建前再次校验排班、服务权限、间隔、预约冲突，避免前端过期数据造成脏写。

### 4. 实习生病历权限

调整 `PrivacyUtils`：

- `apprentice` 不再按“一周内诊疗”泛化授权。
- 只要当前日期落在任一 `internshipDates + 3 天` 窗口内，即拥有只读病历访问。
- 该访问仍不应赋予编辑、删除、收费、库存权限。

### 5. 收费服务权限

- 问诊录入服务项目、收银查看服务项目时，仅展示当前医师 `serviceKeys` 允许的价格表项目。
- 未配置 `serviceKeys` 的老医师，回退为“全部可见”，避免旧数据全失效。

## 前端设计

### 1. 管理端用户管理

在 `AdminView.vue` 用户管理中补齐：

- 医师排序号
- 医师可提供服务多选
- 所属组织
- 组织编号
- 家庭住址
- 实习日期只读表
- “今日实习签到”按钮
- 排班编辑器：每周多时间段列表

### 2. 预约页

预约页改为真正的“排班 + 已预约”视图：

- 周视图保留 7 天列。
- 行按半小时渲染。
- 指定医师时：
  - 上班时段高亮。
  - 已预约时段显示预约卡片。
- 未指定医师时：
  - 单元格显示该时间仍可预约。
  - 可附带可用医师人数/默认匹配医师。

创建预约流程：

- 选择服务
- 选择医师（可空）
- 选择日期
- 加载可用时间
- 提交时由后端最终确认/分配

### 3. 公开预约页

新增匿名页面：

- 选择服务
- 选择医师（可空）
- 选择日期
- 选择时间
- 填写姓名/电话/邮箱/备注
- 提交后自动建档并生成预约

### 4. 病历与收银

- 病历页服务下拉按当前医师服务权限过滤。
- 收银页展示的服务项目仍来自问诊记录，但新增/编辑服务时遵守权限过滤。

## 风险与处理

- 现有用户 `workingHours` 是单段结构：读取时兼容，保存统一写成数组结构。
- 旧医师没有 `serviceKeys`：按“全部服务”兼容。
- 自动建档可能创建重名员工患者：优先按 `linkedUserId` 关联，避免重复。
- 实习权限窗口涉及日期解析：统一按 `Asia/Shanghai` 处理。
